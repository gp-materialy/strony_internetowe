List of 10 bugs in the form:

1 No email field in the newsletter form
The form is for signing up to an email newsletter, yet there is no <input type="email"> field at the top. The "Your Email" field (bug #7) exists but uses the wrong type.
2 Mismatched for attribute in label
The label has for="fullname", but the input has id="name". Clicking the label won't focus the field. Should be for="name" or id="fullname".
3 Password field in a newsletter form
A newsletter doesn't require a password — this field shouldn't exist. Signing up for a newsletter only needs contact information, not account creation.
4 type="time" instead of type="date"
The "Date of Birth" field uses type="time", which lets you pick a time, not a date. It should be type="date".
5 Checkbox instead of radio button for gender
The gender fields use type="checkbox", allowing multiple selections. They should use type="radio" with the same name attribute so only one can be chosen.
6 Swapped min and max values
The rating slider has min="10" and max="1", which is reversed. It should be min="1" max="10".
7 type="number" instead of type="email"
The "Your Email" field uses type="number", which only accepts digits. It should be type="email" so the browser validates the email address.
8 Mismatched for="website" with id="url"
The label has for="website", but the input has id="url". They should match.
9 "Clear" button has type="submit"
The button labelled "Clear Form" actually submits the form (type="submit"). It should have type="reset" or type="button".
10 "Subscribe" button has type="reset"
The "Subscribe to Newsletter" button clears the form instead of submitting it because it has type="reset". It should have type="submit".
