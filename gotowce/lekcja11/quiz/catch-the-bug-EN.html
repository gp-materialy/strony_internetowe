🐛 Catch the Bug in the Form!

Below you'll find a registration form — but someone filled it in WRONG.
How many bugs can you spot in 60 seconds?

  registration-form.html

🐛 Bug: The field is empty, but it has the required attribute.
✅ Fix: Enter a real name, e.g. Anna.
2
🐛 Bug: The address john.com is missing the @ symbol — it's not a valid email.
✅ Fix: Correct it to john@email.com. The type="email" attribute will check this automatically!
3
🐛 Bug: Age -3 is a negative number — that makes no sense!
✅ Fix: Add the attributes min="1" and max="120" to the input.
Enter a real age, e.g. 12.
4
🐛 Bug: The password 123 is only 3 characters — the minimum length is 8.
✅ Fix: Create a longer password, e.g. MyPassword2024!.
The minlength="8" attribute prevents submitting a password that's too short.
5
I accept the terms of service
🐛 Bug: The checkbox is unchecked, but agreement is required (required).
✅ Fix: Check the checkbox. Without this, the form should not submit!
6
0 / 10 characters (minimum 10)
🐛 Bug: The field is empty — 0 characters, but the minimum is 10.
✅ Fix: Write a message. The minlength="10" attribute will enforce at least 10 characters.

🎯 Summary

The form had 6 intentional bugs.
This is exactly how a form works — it has fields, it has rules,
and the browser can check some of them automatically.
